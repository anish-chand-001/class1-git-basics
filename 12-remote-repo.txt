161. WHAT IS A REMOTE REPOSITORY?
=================================

## DEFINITION

A remote repository is a Git repository stored on another machine.

---

## EXAMPLES

✓ GitHub

✓ GitLab

✓ Bitbucket

✓ Company Git Server

↓

↓

✗ No Backup

✗ No Team Collaboration

✗ No Code Review

## WITH REMOTES

162. UNDERSTANDING ORIGIN
=========================

One of Git's most misunderstood terms.

---

## WHAT IS ORIGIN?

Origin is simply a nickname.

Example:

origin
↓
https://github.com/company/project.git

---

## CREATED DURING CLONE

git clone https://github.com/company/project.git

Git automatically creates:

origin

---

## IMPORTANT

Origin is NOT special.

It is only a name.

These would also work:

github

server

production

banana

---

## MENTAL MODEL

origin
=
Default Remote Name

169. FETCHING CHANGES
=====================

## DEFINITION

Download remote updates WITHOUT merging.

---

## COMMAND

git fetch origin

---

## WHAT HAPPENS?

✓ Downloads Changes

✓ Updates References

✓ Does NOT Change Working Code

---

## EXAMPLE

Remote:

A → B → C → D

Local:

A → B → C

After Fetch:

Remote References Updated

Working Directory Unchanged

---

## WHY DEVELOPERS LIKE FETCH

Safe Operation

No Automatic Merge

================================================================================
170. FETCH VS PULL
==================

VERY COMMON INTERVIEW QUESTION

---

## FETCH

git fetch

✓ Downloads Changes

✓ Updates Metadata

✗ No Merge

✗ No Working Directory Changes

---

## PULL

git pull

✓ Downloads Changes

✓ Merges Changes

✓ Updates Working Directory

---

## COMPARISON

Downloads Data        ✓ Fetch   ✓ Pull

Automatic Merge       ✗ Fetch   ✓ Pull

Safe Inspection       ✓ Fetch   ✗ Pull

Updates Branch        ✗ Fetch   ✓ Pull

---

## INDUSTRY RECOMMENDATION

Many senior developers prefer:

git fetch

Review Changes

Then Merge
