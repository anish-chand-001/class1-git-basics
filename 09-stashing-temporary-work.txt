WHY GIT STASH EXISTS
================================================================================

Imagine you're working on:

Feature: JWT Authentication

You've written code for 2 hours.

Suddenly your Team Lead messages:

"Production bug. Fix immediately."

--------------------------------------------------------------------------------
THE PROBLEM
--------------------------------------------------------------------------------

Current work is incomplete.

Cannot commit yet.

Need to switch branches immediately.

--------------------------------------------------------------------------------
WHAT MANY BEGINNERS DO
--------------------------------------------------------------------------------

✗ Create random commits

✗ Copy files manually

✗ Create ZIP backups

✗ Open another VS Code window

--------------------------------------------------------------------------------
WHAT PROFESSIONAL DEVELOPERS DO
--------------------------------------------------------------------------------

git stash

================================================================================
THE REAL PURPOSE OF STASH
================================================================================

Stash is a temporary storage area for unfinished work.

Think:

Working Directory
        |
        v
 Temporary Shelf
    (Stash)
        |
        v
 Continue Later

--------------------------------------------------------------------------------
INSTEAD OF COMMITTING UNFINISHED CODE
--------------------------------------------------------------------------------

✓ Save work temporarily

✓ Clean working directory

✓ Restore later

================================================================================
MENTAL MODEL
================================================================================

Imagine your desk:

Current Work
     |
     v
Boss Arrives
     |
     v
"Need urgent task now!"
     |
     v
Put everything in a drawer
     |
     v
Return later

Drawer = Git Stash

Git Stash works exactly the same way.

================================================================================
111. WHAT IS GIT STASH?
================================================================================

DEFINITION
--------------------------------------------------------------------------------

git stash temporarily saves:

✓ Modified Files

✓ Staged Files

and cleans your working directory.

--------------------------------------------------------------------------------
THINK OF IT AS
--------------------------------------------------------------------------------

Temporary Commit

WITHOUT creating a real commit.

--------------------------------------------------------------------------------
IMPORTANT CHARACTERISTICS
--------------------------------------------------------------------------------

Stash is:

✓ Temporary

✓ Local

✓ Not Shared

--------------------------------------------------------------------------------
WHERE DOES IT EXIST?
--------------------------------------------------------------------------------

Only on your machine.

It is NOT pushed to GitHub.

--------------------------------------------------------------------------------
INDUSTRY USAGE
--------------------------------------------------------------------------------

Common situations:

✓ Urgent Bug Fixes

✓ Branch Switching

✓ Code Reviews

✓ Hotfix Work

✓ Experiments

✓ Context Switching

================================================================================
112. CREATING A STASH
================================================================================

EXAMPLE
--------------------------------------------------------------------------------

authController.js

const login = () => {
    // unfinished work
}

----------------------------------------

authMiddleware.js

// half completed

--------------------------------------------------------------------------------
CHECK STATUS
--------------------------------------------------------------------------------

git status

--------------------------------------------------------------------------------
OUTPUT
--------------------------------------------------------------------------------

Modified:

authController.js

authMiddleware.js

--------------------------------------------------------------------------------
CREATE STASH
--------------------------------------------------------------------------------

git stash

--------------------------------------------------------------------------------
WHAT GIT DOES
--------------------------------------------------------------------------------

Save Changes
      |
      v
Remove Changes
      |
      v
Clean Working Directory

--------------------------------------------------------------------------------
CHECK STATUS AGAIN
--------------------------------------------------------------------------------

git status

--------------------------------------------------------------------------------
OUTPUT
--------------------------------------------------------------------------------

nothing to commit

working tree clean

--------------------------------------------------------------------------------
IMPORTANT
--------------------------------------------------------------------------------

Your code is NOT lost.

It is safely stored in the stash.

--------------------------------------------------------------------------------
WHAT HAPPENS INTERNALLY?
--------------------------------------------------------------------------------

Git creates a:

Hidden Commit

and stores it in:

refs/stash

--------------------------------------------------------------------------------
IMPORTANT INSIGHT
--------------------------------------------------------------------------------

Stash is NOT magic.

It is actually a special Git object.

================================================================================
113. VIEWING STASHES
================================================================================

LIST ALL STASHES
--------------------------------------------------------------------------------

git stash list

--------------------------------------------------------------------------------
EXAMPLE OUTPUT
--------------------------------------------------------------------------------

stash@{0}

stash@{1}

stash@{2}

--------------------------------------------------------------------------------
MORE REALISTIC OUTPUT
--------------------------------------------------------------------------------

stash@{0}: WIP on feature-auth

stash@{2}: WIP on payment-module

--------------------------------------------------------------------------------
UNDERSTANDING STASH NUMBERS
--------------------------------------------------------------------------------

Newest:

stash@{0}

----------------------------------------

Older:

stash@{1}

----------------------------------------

Even Older:

stash@{2}

--------------------------------------------------------------------------------
MENTAL MODEL
--------------------------------------------------------------------------------

Stack

Newest item stays on top.

================================================================================

BEST PRACTICE
--------------------------------------------------------------------------------

Always inspect stash list before applying.

git stash list

================================================================================
VIEWING STASH CONTENT
================================================================================

SUMMARY VIEW
--------------------------------------------------------------------------------

git stash show

--------------------------------------------------------------------------------
DETAILED VIEW
--------------------------------------------------------------------------------

git stash show -p

--------------------------------------------------------------------------------
EXAMPLE OUTPUT
--------------------------------------------------------------------------------

- Removed old auth logic

--------------------------------------------------------------------------------
INDUSTRY USAGE
--------------------------------------------------------------------------------

Very useful before restoring old stashes.

================================================================================
114. APPLYING STASHES
================================================================================

SCENARIO
--------------------------------------------------------------------------------

stash@{0}

contains unfinished JWT work.

--------------------------------------------------------------------------------
RESTORE LATEST STASH
--------------------------------------------------------------------------------

git stash apply

--------------------------------------------------------------------------------
RESTORE SPECIFIC STASH
--------------------------------------------------------------------------------

git stash apply stash@{0}

--------------------------------------------------------------------------------
RESULT
--------------------------------------------------------------------------------

✓ Changes Restored

✓ Stash Still Exists

--------------------------------------------------------------------------------
IMPORTANT
--------------------------------------------------------------------------------

Apply = Copy + Paste

NOT

Cut + Paste

--------------------------------------------------------------------------------
MENTAL MODEL
--------------------------------------------------------------------------------

Apply:

Restore Copy

Keep Original

--------------------------------------------------------------------------------
INDUSTRY USAGE
--------------------------------------------------------------------------------

Useful when same stash may be needed multiple times.

================================================================================
115. POPPING STASHES
================================================================================

Most commonly used stash command.

--------------------------------------------------------------------------------
COMMAND
--------------------------------------------------------------------------------

git stash pop

--------------------------------------------------------------------------------
RESULT
--------------------------------------------------------------------------------

✓ Changes Restored

✓ Stash Deleted

--------------------------------------------------------------------------------
VISUAL
--------------------------------------------------------------------------------

Before:

Stash Exists

----------------------------------------

After:

Changes Restored

Stash Removed

================================================================================
APPLY VS POP
================================================================================

Apply

✓ Restore Changes

✓ Keep Stash

----------------------------------------

Pop

✓ Restore Changes

✗ Delete Stash

================================================================================
INTERVIEW QUESTION
================================================================================

COMMAND             RESTORE        DELETE STASH
------------------------------------------------
git stash apply         ✓                ✗

git stash pop           ✓                ✓

--------------------------------------------------------------------------------
INDUSTRY USAGE
--------------------------------------------------------------------------------

Most developers use:

git stash pop

more often than apply.

--------------------------------------------------------------------------------
DELETE SPECIFIC STASH

git stash drop stash@{1}

--------------------------------------------------------------------------------

stash@{0}

--------------------------------------------------------------------------------
INDUSTRY USAGE
--------------------------------------------------------------------------------

Cleaning old temporary work.

--------------------------------------------------------------------------------

--------------------------------------------------------------------------------

stash@{0}

--------------------------------------------------------------------------------
AFTER

================================================================================
118. PARTIAL STASHING
================================================================================

One of Git's most powerful features.

--------------------------------------------------------------------------------
SCENARIO
--------------------------------------------------------------------------------

Modified Files:

authController.js

paymentController.js

README.md

--------------------------------------------------------------------------------
GOAL
--------------------------------------------------------------------------------

Only stash:

authController.js changes

--------------------------------------------------------------------------------
COMMAND
--------------------------------------------------------------------------------

git stash -p

--------------------------------------------------------------------------------
WHAT HAPPENS?
--------------------------------------------------------------------------------

Git asks:

Stash this change? (y/n)

for each modification.

--------------------------------------------------------------------------------
EXAMPLE
--------------------------------------------------------------------------------

JWT Code
→ y

README Change
→ n

Payment Code
→ n

--------------------------------------------------------------------------------
RESULT
--------------------------------------------------------------------------------

Only selected changes are stashed.

--------------------------------------------------------------------------------
INDUSTRY USAGE
--------------------------------------------------------------------------------

Extremely useful when working on multiple tasks.

================================================================================
WHY SENIOR DEVELOPERS LOVE PARTIAL STASHING
================================================================================

Without Partial Stash:

Everything Gets Stashed

----------------------------------------

With Partial Stash:

Only Selected Changes Get Stashed

Much more control.

================================================================================
119. STASHING UNTRACKED FILES
================================================================================

IMPORTANT LIMITATION
--------------------------------------------------------------------------------

Normal stash:

git stash

does NOT stash:

Untracked Files

--------------------------------------------------------------------------------
EXAMPLE
--------------------------------------------------------------------------------

Created:

newFile.js

Not committed.

Not staged.

--------------------------------------------------------------------------------
RUN
--------------------------------------------------------------------------------

git stash

--------------------------------------------------------------------------------
RESULT
--------------------------------------------------------------------------------

newFile.js remains.

--------------------------------------------------------------------------------
WHY?
--------------------------------------------------------------------------------

Because untracked files are ignored by default.

================================================================================
INCLUDE UNTRACKED FILES
================================================================================

COMMAND
--------------------------------------------------------------------------------

git stash -u

OR

git stash --include-untracked

--------------------------------------------------------------------------------
NOW GIT STASHES
--------------------------------------------------------------------------------

✓ Tracked Files

✓ Untracked Files

--------------------------------------------------------------------------------
INDUSTRY USAGE
--------------------------------------------------------------------------------

Very common when creating:

✓ New Components

✓ New Controllers

✓ New Services

✓ New Files

================================================================================
STASHING IGNORED FILES TOO
================================================================================

Rarely used.

--------------------------------------------------------------------------------
COMMAND
--------------------------------------------------------------------------------

git stash -a

--------------------------------------------------------------------------------
INCLUDES
--------------------------------------------------------------------------------

✓ Tracked Files

✓ Untracked Files

✓ Ignored Files

--------------------------------------------------------------------------------
INDUSTRY NOTE
--------------------------------------------------------------------------------

Most developers never need this.

================================================================================
STASH BEST PRACTICES
================================================================================

RULE #1
--------------------------------------------------------------------------------

Stash is temporary.

Do NOT treat it as long-term storage.

================================================================================

RULE #2
--------------------------------------------------------------------------------

For work lasting days:

Use Branches

NOT Stashes

================================================================================

RULE #3
--------------------------------------------------------------------------------

Name important stashes.

Example:

git stash push -m "JWT middleware work"

================================================================================

RULE #4
--------------------------------------------------------------------------------

Review old stashes regularly.

git stash list

================================================================================

RULE #5
--------------------------------------------------------------------------------

Use:

git stash -u

when working with new files.

================================================================================

RULE #6
--------------------------------------------------------------------------------

Inspect old stashes before applying.

git stash show -p

================================================================================
WHEN NOT TO USE STASH
================================================================================

BAD
--------------------------------------------------------------------------------

3 Weeks Of Work

stored inside stash.

--------------------------------------------------------------------------------
GOOD
--------------------------------------------------------------------------------

30 Minutes

2 Hours

1 Day

Temporary interruption.

--------------------------------------------------------------------------------
IF WORK IS IMPORTANT
--------------------------------------------------------------------------------

Create Branch

Commit Changes

Instead.
