WHY THIS SECTION MATTERS
================================================================================

One of the biggest beginner mistakes is committing files that should never
be tracked by Git.

Examples:

✗ node_modules/
✗ .env
✗ API Keys
✗ Build Files
✗ Log Files
✗ Temporary Files
✗ IDE Settings
✗ Operating System Files

--------------------------------------------------------------------------------
REAL SECURITY RISK
--------------------------------------------------------------------------------

Imagine accidentally pushing:

JWT_SECRET=my_super_secret_key

DATABASE_PASSWORD=admin123

to GitHub.

This has caused real security breaches in companies.

--------------------------------------------------------------------------------
THE PROFESSIONAL RULE
--------------------------------------------------------------------------------

Git SHOULD Track:

✓ Source Code

✓ Configuration Templates

✓ Documentation

✓ Tests

----------------------------------------

Git SHOULD NOT Track:

✗ Secrets

✗ Generated Files

✗ Dependencies

✗ Temporary Files

✗ Machine-Specific Files

--------------------------------------------------------------------------------
THE SOLUTION
--------------------------------------------------------------------------------

.gitignore

================================================================================
101. WHAT IS .gitignore?
================================================================================

DEFINITION
--------------------------------------------------------------------------------

.gitignore is a special file that tells Git:

"Ignore these files and folders.
 Do not track them."

--------------------------------------------------------------------------------
EXAMPLE
--------------------------------------------------------------------------------

.gitignore

node_modules/
.env
dist/

--------------------------------------------------------------------------------
WHAT HAPPENS?
--------------------------------------------------------------------------------

When Git sees:

node_modules/

Git behaves like:

"I know this folder exists.

 I will ignore it.

 I will not track it."

--------------------------------------------------------------------------------
INDUSTRY ANALOGY
--------------------------------------------------------------------------------

Think of .gitignore as:

A Security Guard at the Repository Entrance

----------------------------------------

Allowed:

src/

README.md

package.json

----------------------------------------

Blocked:

node_modules/

.env

logs/

--------------------------------------------------------------------------------
WHY .gitignore EXISTS
--------------------------------------------------------------------------------

Without .gitignore:

✗ Repository Size Increases

✗ Git Operations Become Slower

✗ Sensitive Data Leaks

✗ Unnecessary Files Get Committed

================================================================================
105. IGNORING PATTERNS
================================================================================

Git supports powerful wildcard matching.

--------------------------------------------------------------------------------
IGNORE ALL LOG FILES
--------------------------------------------------------------------------------

*.log

Matches:

server.log

error.log

database.log

--------------------------------------------------------------------------------
IGNORE ALL ENV FILES
--------------------------------------------------------------------------------

.env*

Matches:

.env

.env.local

.env.production

.env.dev

--------------------------------------------------------------------------------
IGNORE ALL ZIP FILES
--------------------------------------------------------------------------------

*.zip

Matches:

backup.zip

project.zip

--------------------------------------------------------------------------------
IGNORE ALL CACHE FILES
--------------------------------------------------------------------------------

*.cache

--------------------------------------------------------------------------------
IGNORE ENTIRE DIRECTORY
--------------------------------------------------------------------------------

logs/

Everything inside logs/ is ignored.

--------------------------------------------------------------------------------
COMMON PATTERN SYMBOLS
--------------------------------------------------------------------------------

1. *

Matches anything.

Example:

*.log

----------------------------------------

2. ?

Matches one character.

Example:

file?.txt

Matches:

file1.txt

file2.txt

Does NOT Match:

file123.txt

----------------------------------------

3. **

Recursive matching.

Example:

**/node_modules/

Matches node_modules anywhere.

================================================================================
106. GLOBAL GIT IGNORE
================================================================================

Some files should be ignored in EVERY repository.

--------------------------------------------------------------------------------
EXAMPLES
--------------------------------------------------------------------------------

.DS_Store

Thumbs.db

.idea/

.vscode/

--------------------------------------------------------------------------------
INSTEAD OF ADDING THEM EVERYWHERE
--------------------------------------------------------------------------------

Create:

~/.gitignore_global

--------------------------------------------------------------------------------
EXAMPLE CONTENT
--------------------------------------------------------------------------------

.DS_Store

Thumbs.db

.vscode/

.idea/

--------------------------------------------------------------------------------
CONFIGURE GLOBAL IGNORE
--------------------------------------------------------------------------------

git config --global core.excludesfile ~/.gitignore_global

--------------------------------------------------------------------------------
RESULT
--------------------------------------------------------------------------------

Git ignores those files in ALL repositories.

--------------------------------------------------------------------------------
INDUSTRY USAGE
--------------------------------------------------------------------------------

Very common among experienced developers.

================================================================================
107. TRACKING PREVIOUSLY IGNORED FILES
================================================================================

IMPORTANT CONCEPT
--------------------------------------------------------------------------------

Suppose:

.env

is ignored.

--------------------------------------------------------------------------------
LATER
--------------------------------------------------------------------------------

You remove:

.env

from .gitignore

--------------------------------------------------------------------------------
NOW
--------------------------------------------------------------------------------

git add .env

works normally.

--------------------------------------------------------------------------------
IMPORTANT RULE
--------------------------------------------------------------------------------

Git only ignores files matching CURRENT rules.

If rule is removed:

File can be tracked again.

--------------------------------------------------------------------------------
INDUSTRY EXAMPLE
--------------------------------------------------------------------------------

Initially:

config.json

ignored

----------------------------------------

Later:

Needs version control

----------------------------------------

Remove ignore rule

Track file normally

================================================================================
108. REMOVING TRACKED FILES
================================================================================

One of the most misunderstood Git topics.

--------------------------------------------------------------------------------
SCENARIO
--------------------------------------------------------------------------------

.env

was already committed.

--------------------------------------------------------------------------------
THEN YOU ADD
--------------------------------------------------------------------------------

.env

to .gitignore

--------------------------------------------------------------------------------
BEGINNER EXPECTATION
--------------------------------------------------------------------------------

Git stops tracking it.

--------------------------------------------------------------------------------
REALITY
--------------------------------------------------------------------------------

Git says:

Already tracking.

Will continue tracking.

--------------------------------------------------------------------------------
WHY?
--------------------------------------------------------------------------------

.gitignore affects ONLY:

Untracked Files

--------------------------------------------------------------------------------
CORRECT SOLUTION
--------------------------------------------------------------------------------

git rm --cached .env

--------------------------------------------------------------------------------
RESULT
--------------------------------------------------------------------------------

✓ Removed From Git

✓ Still Exists On Disk

--------------------------------------------------------------------------------
THEN COMMIT
--------------------------------------------------------------------------------

git commit -m "chore: remove env file from tracking"

--------------------------------------------------------------------------------
INDUSTRY USAGE
--------------------------------------------------------------------------------

Removing:

Secrets

Large Files

Temporary Files

from Git history.

================================================================================
109. CLEANING UNTRACKED FILES
================================================================================

SCENARIO
--------------------------------------------------------------------------------

Project contains:

temp.txt

test.log

random.js

--------------------------------------------------------------------------------
STATUS
--------------------------------------------------------------------------------

git status

--------------------------------------------------------------------------------
OUTPUT
--------------------------------------------------------------------------------

Untracked Files:

temp.txt

test.log

random.js

--------------------------------------------------------------------------------
PREVIEW CLEANUP
--------------------------------------------------------------------------------

git clean -n

--------------------------------------------------------------------------------
OUTPUT
--------------------------------------------------------------------------------

Would remove temp.txt

Would remove test.log

--------------------------------------------------------------------------------
WHY USE -n?
--------------------------------------------------------------------------------

Safe Preview

Nothing gets deleted.

--------------------------------------------------------------------------------
ACTUAL CLEANUP
--------------------------------------------------------------------------------

git clean -f

Deletes files.

--------------------------------------------------------------------------------
DELETE DIRECTORIES TOO
--------------------------------------------------------------------------------

git clean -fd

--------------------------------------------------------------------------------
WARNING
--------------------------------------------------------------------------------

Files are permanently deleted.

No recycle bin.

--------------------------------------------------------------------------------
INDUSTRY USAGE
--------------------------------------------------------------------------------

Cleaning project before builds or releases.

110. PROJECT CLEANUP STRATEGIES
================================================================================

STRATEGY #1
--------------------------------------------------------------------------------

*.log

================================================================================

STRATEGY #2
--------------------------------------------------------------------------------

Keep Repository Small

Avoid:

✗ Videos

✗ ZIP Files

✗ Database Dumps

✗ Large Backups

inside Git.

================================================================================

STRATEGY #3
--------------------------------------------------------------------------------

Never Commit Secrets

BAD:

DATABASE_PASSWORD=123456

JWT_SECRET=secret123

----------------------------------------

GOOD:

.env

and ignore it.

================================================================================

STRATEGY #4
--------------------------------------------------------------------------------

Commit Environment Templates

GOOD:

.env.example

Example:

PORT=

DATABASE_URL=

JWT_SECRET=

(No real values)

--------------------------------------------------------------------------------
INDUSTRY STANDARD
--------------------------------------------------------------------------------

Always commit:

.env.example

Never commit:

.env

================================================================================

STRATEGY #5
--------------------------------------------------------------------------------

Clean Before Committing

Always Run:

git status

before:

git add .

--------------------------------------------------------------------------------
WHY?
--------------------------------------------------------------------------------

Many accidental commits happen because developers skip this step.
